JavaLego es una arquitectura colaborativa que simplifica el desarrollo de aplicaciones empresariales mediante
el uso de un API o especificación Java orientada a servicios, en constante evolución.

"Simplifica tu código"

Ej.: locale de textos y excepciones desde backend a frontend con Angular y Android y servicios y servicios REST usando modelos de datos también.
Seguridad, editores y persistencia de datos con mismos ejemplos.

Categorías de servicios actuales:

- Entornos o contexto de ejecución
- Acceso a datos
- Seguridad
- UI
- Informes
- Servicios de Backend
- Localización de aplicaciones
- Servicios de negocio

Algunos servicios como datos, seguridad, UI o informes requieren de la integración con frameworks, librerías u
otras tecnologías para su implementación.

(imagen de shell o escudo)

Esta integración no modifica nuestro código y nos permite seleccionar la requerida para nuestro proyecto o empresa.
Nuestro código estará desacoplado de la tecnológica y nos permitirá adaptarnos a nuevos cambios tecnológicos preservando nuestro código,
con los beneficios que ello conlleva (migraciones, personal especializado, prevención de errores, preservar la inversión, modificación de TDD, ...) 

Arquitectura no intrusiva. Use sólo los servicios y la tecnología de implementación que necesite.

Implementaciones actuales:

- Acceso a datos: Spring Data, Hibernate
- Seguridad: Shiro, Spring Security
- UI: Vaadin, Android, AngularJS (generación de código)
- Informes: Office, Birt
- Servicios de Backend: Jersey REST
